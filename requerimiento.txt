## 📦 v3.5 — PWA → APK / Play Store

### Diagnóstico actual
- [!] APK muestra barra de URL → es un **WebView wrapper** sin configuración TWA

### Plan de acción

#### Opción A — TWA (Trusted Web Activity) ← RECOMENDADA
- [ ] Configurar `manifest.json`:
  ```json
  "display": "standalone",
  "start_url": "/",
  "scope": "/",
  "theme_color": "#0D0D0D",
  "background_color": "#0D0D0D"
  ```
- [ ] Añadir `Digital Asset Links` en el dominio:
  - Crear archivo `/.well-known/assetlinks.json` con el fingerprint SHA-256 del APK firmado
- [ ] Usar **Bubblewrap CLI** para generar el APK TWA:
  ```bash
  npm i -g @bubblewrap/cli
  bubblewrap init --manifest https://jkirkconell-08.github.io/fitpulse/manifest.json
  bubblewrap build
  ```
- [ ] Subir `.aab` a Google Play Console
- [ ] Verificar con `chrome://flags/#enable-twa` en Android

#### Opción B — Capacitor (más control nativo)
- Requiere migrar a estructura de proyecto Node.js
- Mayor complejidad pero permite wearable APIs en v3

### Validaciones Play Store
- [ ] Lighthouse PWA score ≥ 80
- [ ] HTTPS obligatorio (GitHub Pages ✓)
- [ ] Service Worker con fetch handler ✓
- [ ] Manifest con todos los campos requeridos
- [ ] Ícono 512×512 ✓
